# Skill: Communication Drafting

## Purpose
Help the user draft emails, messages, and other communications efficiently. Match their voice, apply the right structure for the message type, and minimize back-and-forth.

## User Context
User-specific voice and style preferences. Filled in over time as the assistant learns through edits and corrections.

## Approach

### Voice / style match
Default style until the user's preferences are established:
- **Tone:** Direct, friendly, no corporate fluff
- **Length:** Match the recipient's apparent length expectation. Default short.
- **Greetings/sign-offs:** Minimal
- **No filler:** Skip "I hope this finds you well" type intros
- **Active voice:** "I'll send the doc Monday" not "the doc will be sent"

### Common patterns

**Decline politely**
- Acknowledge ask
- State decision
- Brief reason (optional)
- Offer alternative if possible
- No apology marathon

**Request something**
- Lead with the ask
- Provide context (1-2 sentences)
- Specify what you need + by when
- Offer to clarify

**Follow-up**
- Reference prior thread
- Restate the open ask
- Offer easy yes/no path

**Schedule a meeting**
- Purpose (1 sentence)
- Proposed times (3 options)
- Async alternative if applicable

**Negative feedback / disagreement**
- State observation factually
- Distinguish data from interpretation
- Propose path forward
- Avoid "always" / "never"

### Anti-patterns to avoid
- "Just wanted to follow up" → "Following up on X — what's the status?"
- "I think maybe we could perhaps" → "Let's try X."
- Over-apologizing
- Burying the ask in paragraph 4
- Walls of text

## Key Concepts
- **BLUF (Bottom Line Up Front):** state the conclusion or ask in the first sentence
- **One ask per message:** more than one and the recipient picks the easiest
- **Recipient cost > sender cost:** assume the reader will spend 10 seconds, not 5 minutes
- **Asynchronous-first:** offer "I'll proceed unless you object by [date]" to avoid being blocked

## Checklist
1. Ask: what's the goal of the message?
2. Ask: who's the recipient + relationship?
3. Draft with voice match
4. Show 1-2 length variants if appropriate
5. Iterate based on feedback
