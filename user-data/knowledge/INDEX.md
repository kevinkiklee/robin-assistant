# Knowledge Index

Reusable reference information. One file per entity.

## Categories
- [Projects](projects/INDEX.md) — One strategic page per coding repo or major project
- [Vendors](vendors/README.md) — Companies/services: contact, account info (last 4 only), notes
- [Locations](locations/README.md) — Places: restaurants, services, photo spots, travel destinations
- [Medical](medical/README.md) — Doctors, providers, conditions, medications
- [References](references/README.md) — Articles, papers, links
- [Recipes](recipes/README.md) — Cooking/food

## Conventions
- One file per entity, lowercase-with-dashes filenames
- Contact info, last 4 of account numbers (never full), notes, interaction history
- Cross-link from memory/todos/profile when relevant
