## Findings

### F1. Claude Code auto-memory bypass (Claude scenarios 2, 6)

Claude Code's framework-layer auto-memory writes to
`~/.claude/projects/.../memory/`, ignoring AGENTS.md's Local Memory rule.
This is host-level behavior, not solvable from instructions alone.

**Workarounds:**
- Run `claude --bare` to disable auto-memory (also disables CLAUDE.md
  auto-discovery; would need `--append-system-prompt-file=AGENTS.md`).
- Add a Dream-cycle migration step that moves auto-memory entries to
  `user-data/memory/inbox.md` (the rule already prescribes this).
- Use `--disallowedTools "Write(~/.claude/*)"` to block the writes outright.

### F2. `claude --print` skips full session startup (Claude scenario 5)

`claude --print` is one-shot Q&A; doesn't run multi-step session startup.
Tier 1 IS in context (system prompt), but no startup-ritual reads happen.
For scenarios that depend on startup-side-effects (like multi-session
detection), validation needs interactive mode or a different probe.

### F3. Gemini reads files but in cache-suboptimal order (Gemini scenario 1)

Gemini reads all Tier 1 files via `cat` shell commands, but doesn't honor
the AGENTS.md declared order. Cache prefix shifts on every session.
Functional impact: prompt cache slightly less efficient. Not behaviorally
broken — downgraded from HARD to SOFT in the validator.

### F4. Gemini overwrites sessions.md before surfacing sibling (Gemini scenario 5)

Gemini reads sessions.md, sees the sibling row, but in its rush to write
its own session entry it uses `cat > sessions.md <<EOF` (clobbering
overwrite) instead of append. The injected sibling row is lost before
the agent gets to step 2. Then it has nothing to surface.

**Fix candidates:**
- AGENTS.md should explicitly say "APPEND ONLY to sessions.md, never
  overwrite". Currently it says "Append session row" but the verb is
  passive enough that some agents prefer cat > /dev/null heuristics.
- Validate-host could check for any heredoc/cat-overwrite of sessions.md
  and downgrade the failure to soft (agent did try to write, just wrong).

### F5. Trigger-phrase → protocol-fetch works after AGENTS.md fix

Both hosts pass scenario 3 (morning briefing) and scenario 4 (manifest
list) after re-adding the Protocols name list to AGENTS.md. Without it,
Claude Code went on a fishing expedition and never found the protocol.

### F6. AGENTS.md state-path shorthand was wrong

The slim AGENTS.md used `state/sessions.md` (no `user-data/` prefix). The
agent took this literally and tried reading at repo root. Fixed: full
paths everywhere.

### F7. integrations.md was Tier 1 but absent from startup sequence

`token-budget.json` includes `user-data/integrations.md` as Tier 1, but
the slim AGENTS.md didn't mention it in the startup read order. Added
explicitly.
